# 安全审查

对 diff 中的每处改动检查以下攻击面，参考 OWASP Top 10：

1. **注入**：SQL / 命令 / 模板 / LDAP 中存在未参数化的外部输入拼接。字符串拼接构造 SQL 一律报告，即使当前输入"看起来可信"
2. **越权**：新增或修改的接口是否校验了资源归属（当前用户 ↔ 操作的资源）；是否有水平越权（改 id 就能看别人的数据）
3. **敏感信息**：硬编码的密码 / token / 密钥 / 内网地址；日志中输出了密码、身份证、手机号等敏感字段
4. **反序列化与执行**：对不可信数据使用 eval、Function()、pickle、yaml.load（非 safe）、动态 require/import
5. **路径与文件**：用户输入拼接文件路径（路径穿越）、上传文件类型仅靠扩展名判断
6. **认证与会话**：密码明文存储或弱哈希（md5/sha1 无盐）、JWT 未验签、CSRF 保护被移除
7. **SSRF**：服务端用用户提供的 URL 发起请求且无白名单校验

判定标准：
- 存在从外部输入到危险操作的**可达路径** → critical
- 违反纵深防御但当前有上游缓解 → serious
- 硬编码测试凭据、日志敏感信息 → serious

每条发现的 detail 中必须包含一句话攻击路径（谁、通过什么输入、造成什么后果）。
